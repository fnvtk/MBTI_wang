## 0. 目标对齐（一句话）
让后台能够看到：**谁在点什么（中文）→ 谁在分享 → 谁带来了付费 → 单人分润多少**。所有点击目的、按钮含义、测评类型都以**中文**出现。

## 1. 整体分层

| 层 | 位置 | 职责 |
|----|------|------|
| 数据产出 | `miniprogram/**/*.js` | 所有关键按钮 `analytics.track('事件key', { 类型/ID })` |
| 数据聚合 | `api/app/controller/api/Analytics.php` | `POST /api/analytics/events` 批量落库 `analytics_events` |
| 数据翻译 | `api/app/common/AnalyticsEventLabels.php` | 事件 key → 中文名 字典 |
| 统计接口 | `api/app/controller/superadmin/Analytics.php` | `summary / events / user-journey / share-stats` |
| 管理展示 | `admin/src/views/superadmin/MpAnalytics.vue` | 汇总 · 分享榜 · 用户旅程 · 明细 |

## 2. 本轮已完成（✓）

### 2.1 埋点字典（中文） ✓
- 新建 `api/app/common/AnalyticsEventLabels.php`，覆盖：页面浏览、应用启动、测评开始/提交、结果页看全文/分享朋友圈/分享好友/去拍照/解锁完整报告/去完善资料、四测深度 CTA（MBTI/DISC/PDP/SBTI/面相）、推广中心、拍照结果补做问卷、订单、简历、登录/绑手机、支付成功/失败。
- `AnalyticsEventLabels::cn($eventName)` 查字典，未命中回退原 key；`withCn($rows)` 批量补 `eventNameCn` 字段。

### 2.2 统计接口（后端） ✓
- `GET /api/v1/superadmin/analytics/summary?days=&eventName=`  
  事件聚合 + 总数 + **字典 labels**（管理端下拉用）。
- `GET /api/v1/superadmin/analytics/events?days=&page=&pageSize=&eventName=&userId=`  
  事件明细，支持**按事件与用户过滤**；每条附 `eventNameCn`。
- `GET /api/v1/superadmin/analytics/user-journey?userId=&days=`  
  单用户旅程（倒序最近 200 条）。
- `GET /api/v1/superadmin/analytics/share-stats?days=&page=&pageSize=`  
  **分享与邀请 TOP 榜**：
  - `shareCount` = `analytics_events` 中 `share / tap_share_moment / tap_share_friend`
  - `inviteBound` = `distribution_bindings` 按 `inviterId` 聚合
  - `totalCommissionFen` = `distribution_commissions` 按 `inviterId` 聚合
  - 合并后按「分润 → 邀请数 → 分享次数」排序，并补出 `nickname/avatar/phone/enterpriseId/createdAt`。
- 路由注册：`api/route/api.php` 超管分组新增两条。

### 2.3 管理端 UI（MpAnalytics.vue） ✓
- 顶部：统计范围（7/14/30 天）。
- 事件汇总表格：**中文名为主**（粗体），事件 key 为副（灰色），可下拉筛选。
- **分享 & 邀请 · 用户榜**：分享次数 / 邀请人数 / 累计分润，每行「用户旅程」弹窗展示该用户最近 200 条事件（中文名）。
- 最近明细分页（中文 + 原 key）。

## 3. 下一步计划（剩余 · TODO）

### 3.1 补埋点（按优先级）
以下事件字典中 key 已登记，但小程序端需**逐页接入 `analytics.track(...)`**，建议顺序：
1. **首页**（`pages/index/index.js`）：`tap_start_camera`、`tap_questionnaire_home`、`tap_enterprise_entry`
2. **选择测评页**（`pages/test-select/index.js`）：`tap_test_select` + props `{ type: 'mbti' | 'disc' | 'pdp' | 'sbti' }`
3. **答题页**（`pages/test/*`）：`test_start`（已有）、`test_submit`、`test_next`、`test_prev`
4. **结果页共通**：统一把底部三键（看全文 / 朋友圈 / 拍照）的 `track` 加到 `onTapReadFull` / `onTapShareMoment` / `onTapFaceCamera`
5. **推广中心**（`pages/promo/index.js`、`pages/promo/poster.js`、`pages/promo/withdrawals.js`）：`tap_promo_poster`、`tap_promo_withdraw`、`tap_promo_share`
6. **支付**：在 `utils/payment.js` 回调里复用 `reportPayResult(true/false, { type, yuan })`。

> 约定：每个新按钮加入时同步在 `AnalyticsEventLabels.php` 里登记中文名，管理端自动可下拉筛选。

### 3.2 用户旅程增强（P1）
- `UserDetailDialog.vue`（超管 `Users.vue` / `Enterprises.vue` 用户详情弹窗）接入「用户旅程」Tab：直接调 `/superadmin/analytics/user-journey`，展示该用户最近的中文事件时间线，便于排查付费漏斗。
- 参考 soul 永平（`一场soul的创业实验-永平`）的用户画像时间线，做成可筛选事件类型的「像社交轨迹流」。

### 3.3 分享漏斗视图（P2）
在 `MpAnalytics.vue` 加一张「分享漏斗」图（ECharts 漏斗）：
```
page_view(结果页) → tap_share_moment / tap_share_friend → 好友进入(uid 标记) → 好友注册(login_silent_success) → 好友付费(pay_success)
```
对应 SQL（示意）：
```sql
SELECT
  COUNT(DISTINCT CASE WHEN eventName='page_view' AND pagePath LIKE 'pages/result/%' THEN sessionId END) AS 结果页访问,
  COUNT(DISTINCT CASE WHEN eventName IN ('tap_share_moment','tap_share_friend','share') THEN sessionId END) AS 分享动作,
  ...
```
实现顺序：先接口 `/analytics/share-funnel?days=`，再在 UI 画漏斗。

### 3.4 分润成交闭环（P2）
- 邀请落地页 `uid` 写入 `localStorage`，静默登录 `silentLogin` 时带到后端。
- 后端在 `payment/notify` 成功时补一次 `tap_promo_share` 的「成交归因」事件：
  - props `{ inviterId, orderId, amountFen, testType }`。
- 新增后端接口 `/superadmin/analytics/share-attribution?userId=` 查某位用户被邀请的注册→付费链路。

### 3.5 营销链路（跨页串联）
与需求里「页面关联性与成交链路」相关的跨页跳转，已在结果页统一收口：
- 四测结果页 → `tap_deep_service_from_<type>` → `/pages/purchase/index`
- AI 面相结果页 → 三步引导：01 补做四测 · 02 升级深度解读 · 03 分享赚 90% 收益
- 我的 → 推广中心 → 海报/提现

剩余待加的转化位：
- **未完成资料时 tap 事件** → `tap_complete_profile` + props `{ from: 'mbti' | 'disc' | ... }`（目前 `goCompleteProfile` 未埋）
- **付费墙曝光** → `paywall_view` 事件（展示 type、金额）
- **付费墙点击** → `tap_unlock_full` + props `{ type, yuan }`

### 3.6 数据库索引（性能）
`analytics_events` 建议索引：
```sql
ALTER TABLE mbti_analytics_events
  ADD INDEX idx_event_time (eventName, createdAt),
  ADD INDEX idx_user_time (userId, createdAt);
```
（表前缀按 `.env DATABASE_PREFIX`）
