## 2. 关键决策

### 2.1 Tab 顺序与图标
原 3 项（首页 · **拍摄中浮钮** · 我的）→ 新 4 项：

```
首页  神仙AI  [拍摄浮钮]  我的
  0      1        2         3
```

> 保留中间浮钮视觉焦点（拍摄是主要转化入口），「神仙 AI」放在浮钮左侧。
> `hideMiddleFab`（审核模式）场景下，布局自动退化为三项（神仙 AI 仍保留，拍摄隐藏）。

图标：
- `images/ai.png` / `images/ai-active.png`（紫色小星星/sparkle）

### 2.2 聊天页结构（`pages/ai-chat/index`）

```
┌─────────────────────────────────┐
│ 顶部推荐 · 3 篇 MBTI 文章（横滑）   │  ← 调 /api/ai/articles/recommended
│  [封面][标题][来源·一场soul]      │
├─────────────────────────────────┤
│ 我是神仙 AI，你的 MBTI 是 INFP    │  ← 欢迎语（基于 userProfile.mbti）

├─────────────────────────────────┤
```

文章卡片点击 → `webview` 打开原文（soul 的分享链接），同时埋点 `tap_ai_article` `{articleId, source: 'soul'}`（给功能五用）。

### 2.3 AI 对话服务（`AiCallService`）

- 入口：`chat(userId, messages, options)`
- 流程：
  1. 读取 `ai_providers` 表中 `enabled=1 AND visible=1 AND apiKey IS NOT NULL` 的服务商，按 `sortWeight`（新增字段，默认 100）升序
  2. **余额护栏**：任一服务商 `balanceAlertEnabled=1 AND lastBalance < balanceAlertThreshold`，自动降权到最后
  3. 按序尝试调用 OpenAI 兼容 `/chat/completions`，失败或返回 429/401/402/5xx → 切下一家
  4. 全部失败 → 返回降级回答 `"小神仙在喝茶了，稍后再来找我呀～（工程师已收到告警）"` 并写飞书告警
- System Prompt（写入 `AiCallService`）：

  ```
  你是神仙 AI，为神仙团队 MBTI 小程序服务。用户的 MBTI 是 {type}，性格画像：{summary}。
  回答风格：亲切、简短、像朋友；引用 MBTI 理论但不堆术语；适度引导用户去读「一场 soul 创业实验」里的 MBTI 文章。
  禁止：政治 / 医疗诊断 / 投资建议。
  ```

- MBTI 注入：`userProfile.mbtiType` + `utils/mbtiInsights.js` 的四大洞察文本摘要

### 2.4 Soul 文章采集

- 目标接口（以 `https://souladmin.quwanzhi.com/api-docs` 为准，先按约定实现，联调时再校准）：
  - `GET /api/articles?tag=MBTI&limit=10&sort=publishedAt:desc`
- 管理端「Soul 文章推荐」页流程：
  1. 「一键采集最新 10 篇」→ 调后端 `POST /api/superadmin/soul-articles/sync`
  2. 列表展示封面 / 标题 / 来源时间 / MBTI 标签 / 状态
  3. 勾选 → 「推荐」按钮 → 至多 3 篇 `isRecommended=1`（超过弹确认换哪篇）
  4. 小程序实时拉取这 3 篇

### 2.5 欠费预警

复用已有 `balanceAlertEnabled` / `balanceAlertThreshold` / `lastBalance` / `lastBalanceCheckedAt` 字段，新增：

- **定时巡检**：`php think AiBalanceMonitor` 每 60 分钟一次（宝塔计划任务）
  - 调 `queryAllBalances()` → 更新 lastBalance
  - 低于阈值 → `FeishuLeadWebhookService` 发告警到「神仙 MBTI 告警群」
- **前端柔性降级**：`AiCallService` 在 `lastBalance < threshold` 时自动把该服务商降权为 99（沉底）
- **超管看板**：在 `admin/src/views/superadmin/AiConfig.vue` 页顶加一张「AI 健康小条」，实时显示各服务商 余额 / 阈值 / 最近检查时间 / 状态灯

## 3. 数据库变更（P0）

```sql
-- 1) ai_providers 新增 sortWeight（用于优先级）
ALTER TABLE mbti_ai_providers
  ADD COLUMN sortWeight INT NOT NULL DEFAULT 100 AFTER visible;

-- 2) AI 对话表
CREATE TABLE mbti_ai_conversations (
  id INT AUTO_INCREMENT PRIMARY KEY,
  userId INT NOT NULL,
  title VARCHAR(128) DEFAULT '',
  mbtiType VARCHAR(8) DEFAULT '',
  providerId VARCHAR(32) DEFAULT '',
  createdAt INT NOT NULL,
  updatedAt INT NOT NULL,
  INDEX idx_user (userId, updatedAt)
);

-- 3) AI 消息表
CREATE TABLE mbti_ai_messages (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  conversationId INT NOT NULL,
  role ENUM('system','user','assistant') NOT NULL,
  content TEXT NOT NULL,
  tokensIn INT DEFAULT 0,
  tokensOut INT DEFAULT 0,
  providerId VARCHAR(32) DEFAULT '',
  createdAt INT NOT NULL,
  INDEX idx_conv (conversationId, id)
);

-- 4) Soul 采集文章表
CREATE TABLE mbti_soul_articles (
  id INT AUTO_INCREMENT PRIMARY KEY,
  sourceId VARCHAR(64) NOT NULL COMMENT 'soul 侧 id',
  title VARCHAR(255) NOT NULL,
  cover VARCHAR(512) DEFAULT '',
  url VARCHAR(512) NOT NULL,
  summary VARCHAR(512) DEFAULT '',
  tag VARCHAR(32) DEFAULT 'MBTI',
  publishedAt INT NOT NULL,
  isRecommended TINYINT DEFAULT 0,
  recommendedOrder INT DEFAULT 0,
  createdAt INT NOT NULL,
  updatedAt INT NOT NULL,
  UNIQUE KEY uk_source (sourceId),
  INDEX idx_reco (isRecommended, recommendedOrder)
);
```

迁移脚本放 `api/database/migrations/add_ai_chat_and_soul_articles.sql`，部署时一次执行。
