## 一、发现并修复的 Bug ✅

### Bug #1 · 分享统计 / 漏斗：表名字段错误
- **症状**：`/superadmin/analytics/share-stats` 与 `/share-funnel` 里把分润表写成了 `distribution_commissions.amountFen`，而实际数据库表名是 `mbti_commission_records`，金额字段是 `commissionFen`。
- **结果**：超管端「分销 & 邀请 TOP 榜」与漏斗的「累计分润」永远为 0。
- **修复**：`api/app/controller/superadmin/Analytics.php`
  ```php
  // 修正前
  Db::name('distribution_commissions')->sum('amountFen')
  // 修正后
  Db::name('commission_records')->where('commissionFen','>',0)->sum('commissionFen')
  ```
- **附加修正**：`commission_records.createdAt` 是 `int(11)` 整数秒，不是 `datetime`，`$since` 必须改为 `time() - days*86400` 的整数值。

### Bug #2 · 邀请人数重复计算
- **症状**：`distribution_bindings` 同一个 `inviteeId` 可能因 scope / enterpriseId 不同出现多行，`COUNT(*)` 会重复。
- **修复**：改为 `COUNT(DISTINCT inviteeId)` + `WHERE status='active'`。

### Bug #3 · Payment.notify 归因用了过期 binding
- **症状**：查询 `distribution_bindings` 没有 `status` / `expireAt` 过滤，导致归因事件的 `inviterId` 可能指向失效绑定。
- **修复**：`api/app/controller/api/Payment.php`
  ```php
  Db::name('distribution_bindings')
    ->where('inviteeId', $userId)
    ->where('status', 'active')
    ->where('expireAt', '>', time())
    ->order('id', 'desc')
    ->find();
  ```

### Bug #4 · 漏斗「好友登录」永远为 0
- **症状**：漏斗第 3 段 `login_silent_success` 事件在字典里登记了，但 `miniprogram/app.js` 没上报过。
- **修复**：`silentLogin()` 成功/失败都补埋：
  - 成功：`login_silent_success` + props `{ userId, hasEnterprise, invited }`
  - 失败：`login_silent_fail` + props `{ reason: non_200 | network | wx_login }`

### Bug #5 · DISC/PDP/SBTI 结果页 onShow 没刷新 Journey
- **症状**：用户完善资料后回到结果页，步骤条 step1 仍然是未解锁态。
- **修复**：四个结果页的 `onShow` 统一调用 `this._syncJourney()`。

### Bug #6 · SBTI 付费墙曝光未同步 Journey
- **症状**：`initPayInfoFromRuntime` 回调里只 setData payInfo，未触发 Journey 重算。
- **修复**：SBTI/DISC/PDP/MBTI 四份同步补 `this._syncJourney()`。
