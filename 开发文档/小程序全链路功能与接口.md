# 小程序全链路：页面 → 接口 → 数据（梳理）

> 路由以 `miniprogram/app.json` 为准；接口前缀为 `app.globalData.apiBase`（生产示例 `https://mbtiapi.quwanzhi.com`）。数据落库以 PHP API + MySQL 为准（表结构见 `api/mbti_data.sql` 等）。

## Tab 与首页动线

| 页面 | 作用 | 主要接口 / 存储 |
|------|------|-----------------|
| `pages/index/index` | 首页入口 | `/api/auth/me`；分销 scene 参数进 `enterprise` |
| `pages/index/camera` | Tab「查看报告」：面相入口 | 跳转 `upload` 或 `result` |
| `pages/index/upload` | 三张照片上传 | `POST /api/upload/image`；`wx.setStorageSync('aiPhotos', urls)` |
| `pages/index/result` | AI 面相分析结果 | `POST /api/analyze`；付费 `utils/payment` → `/api/payment/*` |
| `pages/profile/index` | Tab「我的」 | `/api/test/recent`、`/api/distribution/stats` |

## 问卷测评

| 页面 | 作用 | 结果落地 |
|------|------|----------|
| `pages/test-select/index` | 选 MBTI / DISC / PDP | `navigateTo` 各测试页 |
| `pages/test/mbti` | MBTI 答题 | 本地 `mbtiResult` + `app.saveTestResult('mbti')` → `redirectTo` `result/mbti` |
| `pages/test/disc` | DISC | `discResult` → `result/disc` |
| `pages/test/pdp` | PDP | `pdpResult` → `result/pdp` |
| `pages/result/mbti` 等 | 结果展示 / 付费解锁 | 详情 `GET /api/test/detail`；付费同 `payment.js` |

## 深度服务与支付

| 页面 | 作用 | 接口 |
|------|------|------|
| `pages/purchase/index` | 深度服务 Tab、类目价目 | `GET /api/config/deep-pricing`；购买 `POST /api/payment/create`；线索 `POST /api/crm/report` |
| `pages/recharge/index` | 企业充值相关 | `/api/enterprise/bind` 等 |

## 企业与简历

| 页面 | 作用 | 接口 |
|------|------|------|
| `pages/enterprise/index` | 企业绑定 / 入口 | `POST /api/enterprise/bind` |
| `pages/enterprise/resume-history` | 我的简历 | `GET/POST/... /api/enterprise/resume-uploads*` |

## 用户与推广

| 页面 | 作用 | 接口 |
|------|------|------|
| `pages/user-profile/index` | 资料、头像、手机 | `PUT /api/auth/wechat/profile`、`getPhoneNumber` → `POST /api/auth/wechat/phone` |
| `pages/phone-auth/index` | 手机授权页 | 同 phone 接口 |
| `pages/promo/*` | 推广中心 / 海报 / 提现 | `/api/distribution/*` |
| `app.js` | 静默登录、分销 bind | `POST /api/auth/wechat`、`POST /api/distribution/bind` |

## 全局工具

- `utils/request.js`：统一 Bearer、401 清登录态。
- `utils/payment.js`：创建订单、调起支付、轮询 `/api/payment/query`。
- `utils/phoneAuth.js`：`hasPhone`、`bindPhoneByCode`、`isProfileComplete`（**已绑手机即视为满足门禁**，避免头像未上传死循环）。
